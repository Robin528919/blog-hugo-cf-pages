# 小红书自动化发布 - 实施清单

> 从研究到上线的完整检查清单

**使用方法**: 根据您选择的方案，逐项打勾完成

---

## 前期准备（所有方案通用）

### 账户与资质
- [ ] 小红书账户已创建
- [ ] 账户已完成实名认证
- [ ] 账户符合发布条件（无违规记录）
- [ ] 已获取小红书用户 ID

### 内容准备
- [ ] Hugo 博客已建立并正常运行
- [ ] 有 3-5 篇待发布的文章
- [ ] 每篇文章有相应的配图（至少 3 张）
- [ ] 配图使用 3:4 竖版尺寸（1080×1440px）
- [ ] 文章已进行小红书格式审查（字数、标签等）

### 技术环境
- [ ] Python 3.8+ 已安装
- [ ] Chrome/Chromium 浏览器已安装
- [ ] Git 已配置（用于 GitHub Actions）
- [ ] GitHub 仓库已创建并配置

## 方案 1: Wechatsync（推荐首先尝试）

### 安装阶段
- [ ] Chrome 浏览器已更新到最新版本
- [ ] 访问 https://www.wechatsync.com/ 并安装扩展
- [ ] 扩展已添加到 Chrome
- [ ] 扩展权限已授予（访问所有网站）

### 账户配置
- [ ] 微信公众号已登录（mp.weixin.qq.com）
- [ ] 小红书已登录（xiaohongshu.com）
- [ ] 目标发布平台已全部登录（知乎、掘金等）
- [ ] 浏览器 Cookie 和登录态已保存

### 功能测试
- [ ] 发布一篇测试文章到微信公众号
- [ ] 打开发布的微信文章页面
- [ ] 点击 Wechatsync 扩展图标
- [ ] 可以看到支持的平台列表（包括小红书）
- [ ] 选择"小红书"进行同步
- [ ] 等待 1-2 分钟
- [ ] 登录小红书检查草稿箱
- [ ] 验证文章是否已同步到小红书
- [ ] 检查格式转换是否正确
- [ ] 检查图片是否全部迁移

### 发布工作流确认
- [ ] 在小红书草稿箱中找到同步的文章
- [ ] 检查标题、正文、图片是否完整
- [ ] 根据需要调整内容（删除微信特有内容等）
- [ ] 添加合适的标签和话题
- [ ] 设置是否允许评论
- [ ] 完成发布

### 文档记录
- [ ] 记录 Wechatsync 同步过程中的问题
- [ ] 更新 Hugo 文章 Front Matter（添加发布状态）
- [ ] 创建发布日志（手动或脚本化）

### 优化阶段（可选）
- [ ] 分析小红书发布后的数据（浏览量、互动）
- [ ] 根据反馈调整内容策略
- [ ] 建立定期发布计划

**完成标准**: 已成功通过 Wechatsync 发布至少 1 篇文章到小红书

## 方案 2: Playwright MCP（完全自动化）

### 环境准备
- [ ] Python 3.8+ 已验证 (`python --version`)
- [ ] pip 已更新 (`pip install --upgrade pip`)
- [ ] 虚拟环境已创建 (`python -m venv venv`)
- [ ] 虚拟环境已激活

### 项目克隆与安装
- [ ] 克隆 Redbook-Search-Comment-MCP 项目
  ```bash
  git clone https://github.com/chenningling/Redbook-Search-Comment-MCP2.0
  cd Redbook-Search-Comment-MCP2.0
  ```
- [ ] 进入项目目录
- [ ] 安装依赖 (`pip install -r requirements.txt`)
- [ ] 验证 Playwright 已安装 (`pip list | grep playwright`)
- [ ] 下载浏览器驱动 (`playwright install chromium`)

### 配置阶段
- [ ] 找到 `config.json` 或 `.env` 配置文件
- [ ] 创建 `.env` 文件（如不存在）
- [ ] 添加小红书账户信息：
  ```
  XHS_USERNAME=your_phone_or_email
  XHS_PASSWORD=your_password
  ```
- [ ] 验证配置文件权限（不应被 Git 跟踪）
- [ ] 在 `.gitignore` 中添加 `.env`

### 功能测试
- [ ] 运行项目的登录测试脚本
- [ ] 浏览器自动打开小红书
- [ ] 等待自动登录完成
- [ ] 如需手动验证，在提示时完成
- [ ] 验证登录成功（页面显示已登录状态）
- [ ] 关闭浏览器

### 自动发布测试
- [ ] 准备一篇测试文章（Markdown 格式）
- [ ] 运行发布脚本（如存在）
- [ ] 观察浏览器自动化过程
- [ ] 验证文章是否成功发布
- [ ] 检查小红书后台是否显示新发布的笔记
- [ ] 记录任何错误或警告信息

### 脚本开发
- [ ] 创建 `scripts/xiaohongshu/` 目录
- [ ] 复制或编写 `hugo_to_xhs.py` 脚本
- [ ] 脚本应包含以下功能：
  - [ ] 读取 Hugo Markdown 文件
  - [ ] 解析 Front Matter（标题、标签、日期）
  - [ ] 提取文章配图
  - [ ] 处理内容格式（移除 Markdown 语法）
  - [ ] 调用发布函数
  - [ ] 记录发布状态
- [ ] 添加错误处理和日志输出
- [ ] 添加 dry-run 模式（预览而不实际发布）

### 集成 GitHub Actions
- [ ] 创建 `.github/workflows/` 目录（如不存在）
- [ ] 创建 `xiaohongshu-auto-publish.yml` workflow 文件
- [ ] 配置 workflow 触发条件（push 到 content/posts/）
- [ ] 添加 Python 环境配置
- [ ] 添加依赖安装步骤
- [ ] 添加脚本执行步骤
- [ ] 在 GitHub 项目设置中添加 Secrets：
  - [ ] `XHS_USERNAME`
  - [ ] `XHS_PASSWORD`
- [ ] 测试 workflow（推送一个小改动）
- [ ] 检查 GitHub Actions 日志（Actions 标签页）
- [ ] 验证小红书是否收到发布

### 监控与维护
- [ ] 设置 GitHub Actions 失败通知
- [ ] 创建发布状态日志文件（如 `xhs_published.json`）
- [ ] 建立定期检查机制（每周检查 1 次）
- [ ] 更新 `selectors.py` 如果页面 DOM 变更

### 安全加固
- [ ] 确保密码不会被提交到 Git
- [ ] 检查 `.gitignore` 包含 `.env` 文件
- [ ] 检查 GitHub Secrets 设置正确
- [ ] 定期更新依赖包 (`pip install --upgrade -r requirements.txt`)
- [ ] 考虑使用 GitHub Actions 秘钥轮换策略

**完成标准**: 已成功通过 GitHub Actions 自动发布至少 1 篇文章，无需手动干预

## 方案 4: CLI 工具（快速验证）

### 安装
- [ ] 安装 xiaohongshu-cli：
  ```bash
  pip install xiaohongshu-cli
  ```
- [ ] 验证安装成功：
  ```bash
  xhs --version
  ```

### 基础测试
- [ ] 搜索笔记：
  ```bash
  xhs search "技术写作"
  ```
- [ ] 获取笔记详情：
  ```bash
  xhs get <note-id>
  ```

### 发布测试
- [ ] 准备测试图片
- [ ] 运行发布命令：
  ```bash
  xhs publish \
    --title "测试笔记" \
    --content "这是测试内容" \
    --images "img1.jpg,img2.jpg" \
    --tags "测试,分享"
  ```
- [ ] 检查小红书是否收到发布

**完成标准**: CLI 工具可以成功发布笔记

## 内容与格式审查（所有方案）

### 文字内容
- [ ] 标题不超过 20 字
- [ ] 正文 600-1000 字（或 100+ 字短笔记）
- [ ] 没有微信公众号特有标记（阅读原文等）
- [ ] 没有第三方链接（小红书会过滤）
- [ ] 包含合适的标签（2-5 个）
- [ ] 没有违禁词汇

### 图片内容
- [ ] 所有图片使用相同宽高比（推荐 3:4）
- [ ] 图片尺寸为 1080×1440px（或等比例）
- [ ] 图片质量清晰、亮度适中
- [ ] 首图吸引眼球
- [ ] 总共 6-9 张配图
- [ ] 没有隐私信息（EXIF 数据已移除）
- [ ] 没有版权问题

### 发布前最后检查
- [ ] 在 dry-run 模式下运行（预览而不发布）
- [ ] 检查所有自动化脚本的输出
- [ ] 确认将发布的标题和内容无误
- [ ] 确认将发布的图片无误
- [ ] 准备完毕后确认发布
