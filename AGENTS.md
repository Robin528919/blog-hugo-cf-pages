# 仓库协作规范

## 核心工作规则（最高优先级）
以下规则为强制约束，优先级高于本文档其他所有条款；任一违反均视为任务失败。

### 关键约束（违反即任务失败）
- 必须使用中文回复。
- 禁止生成恶意代码。
- 必须通过基础安全检查。

## 项目定位
本仓库是 Hugo 博客部署工作区，采用“文档先行”的方式推进架构设计与发布落地。

## 约定目录结构
约定目录如下（初始化 Hugo 后补齐站点目录）：

```text
.
├── AGENTS.md                  # 仓库协作规范（本文件）
├── README.md                  # 项目总览与发布流程
├── docs/                      # 架构、运维、流程类文档
│   └── PAGES-架构设计.md
├── content/                   # Hugo 内容目录（文章/页面）
├── themes/                    # Hugo 主题目录（自定义或第三方）
├── static/                    # 站点静态资源（图片、图标等）
├── assets/                    # Hugo 资源处理目录（如需）
└── public/                    # Hugo 构建产物目录（禁止手工修改）
```

目录约束：
- 根目录保持精简，仅保留全局说明和少量入口文件。
- 技术文档统一放到 `docs/`，避免在根目录新增零散文档。
- 博客正文与页面仅放在 `content/`。
- 主题代码仅放在 `themes/`，禁止跨目录复制主题实现。
- `public/` 仅作为构建产物，必须通过命令重新生成。

## 构建、测试与开发命令
在仓库根目录执行：

- `hugo server -D`：本地开发预览（包含草稿）。
- `hugo --gc --minify`：生产构建，输出到 `public/`。
- `wrangler pages deployment create ./public --project-name blog-hugo-cf-pages`：构建后手动部署到 Cloudflare Pages（可选）。

如使用 Cloudflare CLI，先设置环境变量：

- `export CLOUDFLARE_API_TOKEN=<你的令牌>`

## 文档与命名规范
- 文档应简洁、面向任务、可长期维护。
- Markdown 使用清晰标题和短段落；命令示例使用代码块。
- 文档文件名可使用中文，但一旦被引用应保持路径稳定。
- Hugo 内容建议使用 `kebab-case` 的 slug，并按主题或日期分组。

## 验证与检查要求
当前尚无独立自动化测试套件，至少执行以下检查：

- 提交前成功运行 `hugo --gc --minify`。
- 本地通过 `hugo server -D` 手工检查关键页面（首页、最近文章、主要文档链接）。
- 确认未提交令牌、密钥等敏感信息。

## 提交与合并请求规范
当前提交信息建议使用轻量前缀：

- `docs: ...`：文档变更
- `chore: ...`：仓库与初始化类变更

合并请求要求：

- 说明变更范围与影响文件。
- 有关联任务时附上链接。
- 涉及页面展示变更时附 Cloudflare Pages 预览地址或截图。
- 保持单一主题，避免把无关重构与文档修改混在同一请求中。

## 安全与配置建议
- 禁止提交 API Token、账号凭据等敏感信息。
- 敏感配置使用环境变量（如 `CLOUDFLARE_API_TOKEN`）。
- 使用 `main` 作为生产分支，`develop` 或 PR 分支作为预览分支。
